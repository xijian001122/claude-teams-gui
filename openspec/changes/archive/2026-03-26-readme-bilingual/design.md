## Context

当前 README.md 结构简单，仅包含英文的快速开始和一些基本命令。项目面向中文用户，但文档缺乏中文支持。新用户需要理解：
1. 项目是什么，解决了什么问题
2. 如何快速上手
3. 各项功能如何使用

## Goals / Non-Goals

**Goals:**
- 提供清晰的项目背景和价值说明
- 中英双语内容（中文为主，英文辅助）
- 包含完整的安装、配置、使用教程
- 包含常见问题解答
- 保持文档可维护性

**Non-Goals:**
- 不翻译 docs/ 下的其他文档（保持独立维护）
- 不修改代码或添加功能
- 不包含 API 文档（已在 docs/development.md）

## Decisions

### 双语结构选择

**决定**: 中文在前，英文在后（中文为主）

**理由**:
- 项目主要面向中文用户
- 便于中文用户快速理解
- 英文作为辅助，方便国际贡献者

### 章节结构

```
1. 项目标题和简介（中/英）
2. 项目背景和使用场景
3. 主要特性
4. 版本信息
5. 快速开始
6. 详细安装配置教程
7. 功能使用说明
8. 常见问题（FAQ）
9. 文档链接
10. 架构图
11. 开发指南
12. 版本管理
13. License
```

### FAQ 内容

**决定**: 包含以下常见问题：
- 如何修改端口？
- 如何启用/禁用桌面通知？
- 如何切换主题？
- 如何查看历史消息？
- 数据存储在哪里？

## Risks / Trade-offs

- [Risk] 双语内容维护成本增加
  - **Mitigation**: 保持结构简单，中文表达完整，英文简洁翻译
- [Trade-off] README 可能变得过长
  - **Mitigation**: 使用折叠区块（<!-- details -->）或链接到 docs/ 详情

## Migration Plan

2. 验证格式和链接正确性
3. 提交并推送
