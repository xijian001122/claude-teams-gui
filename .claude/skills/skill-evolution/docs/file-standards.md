# 文件规范详解

## 概述

本文档详细说明技能系统中各类文件的规范和要求。

## SKILL.md 规范

### 必需内容

#### 1. YAML Frontmatter
```yaml
---
name: skill-name              # kebab-case，最多 64 字符
description: 技能描述          # 说明何时使用此技能
---
```

**要求**：
- ✅ 必须以 `---` 开头和结尾
- ✅ name 使用 kebab-case
- ✅ description 简洁明了（1-2 句话）

#### 2. 技能标题
```markdown
# 技能名称

**作用**: 简要说明技能的用途

**触发关键词**: 关键词1、关键词2、关键词3
```

#### 3. 核心规范速查
```markdown
## 核心规范速查

| 规范 | 说明 | 详细文档 |
|------|------|---------|
| 规范1 | 简要说明 | [docs/topic1.md](docs/topic1.md) |
| 规范2 | 简要说明 | [docs/topic2.md](docs/topic2.md) |
```

#### 4. 快速检查清单
```markdown
## 快速检查清单

- [ ] 检查项1
- [ ] 检查项2
- [ ] 检查项3
```

#### 5. 详细文档索引
```markdown
## 详细文档索引

### 核心流程
- [主题1](docs/topic1.md) - 详细说明
- [主题2](docs/topic2.md) - 详细说明

### 最佳实践
- [最佳实践](docs/best-practices.md) - 开发建议
```

#### 6. 相关技能
```markdown
## 相关技能

- **related-skill-1**: 相关技能说明
- **related-skill-2**: 相关技能说明
```

#### 7. 页脚信息
```markdown
---

**最后更新**: YYYY-MM-DD
**维护者**: 开发团队
```

### 文件大小限制
- **最大行数**：500 行
- **超出处理**：将详细内容移到 docs/，代码模板移到 QUICK_REF.md

### 链接规范
- ✅ 使用相对路径：`[文档](docs/topic.md)`
- ❌ 避免绝对路径：`[文档](/absolute/path/topic.md)`
- ✅ 锚点链接：`[章节](#section-name)`

## QUICK_REF.md 规范

### 内容组织

#### 1. 文档标题
```markdown
# 快速参考

本文档提供常用代码模板和速查表。
```

#### 2. 代码模板
```markdown
## 模板1：基础模板

```java
// 完整可运行的代码示例
public class Example {
    private String field;

    public String getField() {
        return field;
    }
}
```

👉 [详细说明](docs/topic1.md)
```

**要求**：
- ✅ 代码完整可运行
- ✅ 添加必要注释
- ✅ 标注代码语言
- ✅ 链接到详细文档

#### 3. 命令速查
```markdown
## 常用命令

| 命令 | 说明 |
|------|------|
| 命令1 | 说明1 |
| 命令2 | 说明2 |
```

#### 4. 链接到详细文档
```markdown
## 相关文档

- [详细主题1](docs/topic1.md)
- [详细主题2](docs/topic2.md)
```

### 文件大小
- **限制**：无严格限制
- **建议**：保持在 1000 行以内，便于维护

## docs/ 文档规范

### 文件命名
- **格式**：kebab-case
- **示例**：`controller-layer.md`、`best-practices.md`
- **避免**：驼峰命名、空格、特殊字符

### 文件大小
- **推荐范围**：200-400 行
- **最小**：100 行（避免过小文件）
- **最大**：400 行（避免过大文件）

### 文档结构

#### 1. 标题和概述
```markdown
# 主题标题

## 概述

简要说明本主题的内容和目的。
```

#### 2. 核心概念
```markdown
## 核心概念

### 概念1
详细说明概念1。

### 概念2
详细说明概念2。
```

#### 3. 使用示例
```markdown
## 使用示例

### 示例1：基础用法
```java
// 代码示例
```

**说明**：示例说明

### 示例2：高级用法
```java
// 代码示例
```

**说明**：示例说明
```

#### 4. 最佳实践
```markdown
## 最佳实践

1. **实践1**：说明
2. **实践2**：说明
3. **实践3**：说明
```

#### 5. 常见问题
```markdown
## 常见问题

**解决方案**：解决步骤

```

#### 6. 相关文档
```markdown
## 相关文档

- [相关主题1](topic1.md)
- [相关主题2](topic2.md)
```

### 内容要求
- ✅ 单一主题
- ✅ 结构清晰
- ✅ 代码示例完整
- ✅ 链接有效

## CHANGELOG.md 规范

### 格式规范

遵循 [Keep a Changelog](https://keepachangelog.com/) 规范：

```markdown
# Changelog

## [版本号] - YYYY-MM-DD
### Added
- 新增内容1
- 新增内容2

### Changed
- 变更内容1
- 变更内容2

### Fixed
- 修复内容1
- 修复内容2

### Removed
- 移除内容1
- 移除内容2
```

### 版本号规则

遵循 [语义化版本](https://semver.org/) 规范：

- **Major（主版本）**：不兼容的 API 修改
  - 示例：删除文档、重大重构
  - 版本号：1.0.0 → 2.0.0

- **Minor（次版本）**：向下兼容的功能性新增
  - 示例：新增文档、新增规范
  - 版本号：1.0.0 → 1.1.0

- **Patch（修订号）**：向下兼容的问题修正
  - 示例：修正错别字、修复链接
  - 版本号：1.0.0 → 1.0.1

### 变更类型

| 类型 | 说明 | 示例 |
|------|------|------|
| Added | 新增功能 | 新增文档、新增规范 |
| Changed | 变更内容 | 更新文档、修改规范 |
| Fixed | 修复问题 | 修复链接、修正错误 |
| Removed | 移除内容 | 删除文档、移除规范 |

## 通用规范

### Markdown 格式

#### 标题层级
```markdown
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（详细说明）
```

**要求**：
- ✅ 每个文档只有一个一级标题
- ✅ 标题层级连续（不跳级）
- ✅ 标题简洁明了

#### 代码块
```markdown
```java
// 代码示例
public class Example {
    // 实现代码
}
```
```

**要求**：
- ✅ 标注代码语言
- ✅ 代码完整可运行
- ✅ 添加必要注释

#### 表格
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
```

**要求**：
- ✅ 表头清晰
- ✅ 对齐美观
- ✅ 内容简洁

#### 列表
```markdown
- 无序列表项1
- 无序列表项2

1. 有序列表项1
2. 有序列表项2
```

**要求**：
- ✅ 层级清晰
- ✅ 缩进一致
- ✅ 内容简洁

#### 链接
```markdown
[链接文本](相对路径)
[链接文本](相对路径#锚点)
```

**要求**：
- ✅ 使用相对路径
- ✅ 链接文本描述清晰
- ✅ 验证链接有效性

### 文件编码
- **编码**：UTF-8（无 BOM）
- **换行符**：LF（Unix 风格）

### 文件权限
- **权限**：644（rw-r--r--）
- **目录权限**：755（rwxr-xr-x）
