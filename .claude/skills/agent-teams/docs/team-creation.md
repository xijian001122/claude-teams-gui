# 团队创建

## 概述

本文档详细说明如何创建 Agent Teams 和 spawn 团队成员。

## 创建团队

### 使用 TeamCreate 工具

```javascript
TeamCreate({
  team_name: "claude-teams-gui",           // 团队名称（必需）
  agent_type: "team-lead",         // 团队领导类型（可选）
  description: "开发测试协作团队" // 团队描述（可选）
})
```

**参数说明**:
- `team_name`: 团队名称，使用 kebab-case（如 claude-teams-gui）
- `agent_type`: 团队领导的类型，默认为 "team-lead"
- `description`: 团队的描述，说明团队的用途

### 团队配置文件

团队创建后，会生成配置文件：`~/.claude/teams/{team-name}/config.json`

## Spawn 团队成员

### 使用 Agent 工具

```javascript
Agent({
  name: "developer",                   // 成员名称（必需）
  description: "开发者 - 负责功能开发",  // 简短描述（必需）
  prompt: "详细的角色说明和职责...",       // 详细提示（必需）
  subagent_type: "general-purpose",       // Agent 类型（可选）
  model: "kimi-k2.5",                        // 模型（可选，默认继承父进程）
  team_name: "claude-teams-gui"                   // 团队名称（必需）
})
```

**参数说明**:
- `name`: 成员名称，用于标识和通信
- `description`: 简短描述（3-5 词），用于工具调用
- `prompt`: 详细的角色说明，包括职责和工作方式
- `subagent_type`: Agent 类型，决定可用工具
- `model`: 模型别名（`"kimi-k2.5"`, `"glm-5"`）
- `team_name`: 所属团队名称

## Agent 类型选择

### general-purpose（通用型）

**可用工具**: 所有工具（Read、Edit、Write、Bash、Glob、Grep 等）

**适用场景**:
- 功能开发
- Bug 修复
- 测试编写
- 代码重构

### Explore（探索型）

**可用工具**: 只读工具（Glob、Grep、Read）

**适用场景**:
- 代码探索
- 架构研究
- 问题调查

### Plan（规划型）

**可用工具**: 只读工具 + 规划工具

**适用场景**:
- 架构设计
- 实现规划
- 技术方案设计

## 团队成员角色配置

| 角色 | 名称 | 模型 | 职责 |
|------|------|------|------|
| 前端开发者 | frontend-dev | kimi-k2.5 | 前端功能开发、UI实现 |
| 后端开发者 | backend-dev | glm-5 | 后端API开发、数据库操作 |
| 测试者 | tester | glm-5 | 测试验证、质量保证 |
| Bug修复 | bug-fixer | glm-5 | 问题修复、调试排错 |

**模型说明**:
- **kimi-k2.5**: 高性能模型，适合复杂前端逻辑开发
- **glm-5**: 平衡性能，适合后端、测试和修复任务

## 最佳实践

### 1. 团队命名

- 使用 kebab-case（如 claude-teams-gui）
- 名称要有意义，反映团队用途

### 2. 成员命名

- 使用简短、清晰的名称（如 developer、tester、bug-fixer）
- 名称要反映成员角色

### 3. Agent 类型选择

- 需要修改代码 → general-purpose
- 只需要研究 → Explore
- 只需要规划 → Plan

### 4. 模型选择

- 前端复杂开发任务 → kimi-k2.5
- 后端、测试、修复任务 → glm-5
