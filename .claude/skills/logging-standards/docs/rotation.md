# 文件轮转机制

## 轮转策略

日志系统支持两种轮转方式：

### 1. 按日期轮转

**触发条件**：
- 服务启动时检测到日志文件修改日期不是今天
- 运行时跨天（00:00）

**行为**：
- 将当前日志移动到日期目录
- 创建新的空日志文件

### 2. 按大小轮转

**触发条件**：
- 日志文件大小超过 `maxSize` MB

**行为**：
- 将当前日志移动到日期目录（带序号）
- 创建新的空日志文件

## 目录结构

```
~/.claude-chat/logs/
├── console.log           # 当前日志（所有级别）
├── info.log              # 当前日志（INFO+）
├── error.log             # 当前日志（ERROR only）
├── 2026-03-28/           # 历史日志（按日期）
│   ├── console-001.log
│   ├── console-002.log   # 同一天多次轮转
│   ├── info-001.log
│   └── error-001.log
├── 2026-03-29/
│   └── ...
└── 2026-03-30/           # 今天的历史（如果已有轮转）
    └── ...
```

## 配置参数

```typescript
interface LogConfig {
  maxSize: number;  // 单文件最大大小（MB）
  maxDays: number;  // 保留天数
}
```

### 默认值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `maxSize` | 10 | 10 MB |
| `maxDays` | 7 | 7 天 |

## 轮转流程

### 启动时检查

```
服务启动
    ↓
检查 console.log 存在？
    ↓ 是
读取文件修改时间（mtime）
    ↓
mtime 不是今天？
    ↓ 是
执行日期轮转
    ↓
打开新日志流
```

### 运行时检查

```
每次写日志
    ↓
检查当前日期 != 记录日期？
    ↓ 是
执行日期轮转
    ↓
检查文件大小 >= maxSize？
    ↓ 是
执行大小轮转
```

## 过期清理

每天启动时检查历史日志目录：

1. 计算截止日期 = 今天 - maxDays
2. 遍历日志目录
3. 删除早于截止日期的目录

**示例**：
- 今天：2026-03-30
- maxDays：7
- 截止日期：2026-03-23
- 删除：2026-03-22 及更早的目录

## 最佳实践

### 1. 合理设置 maxSize

```typescript
// 开发环境：较小，便于查看
maxSize: 5

// 生产环境：较大，减少轮转频率
maxSize: 50
```

### 2. 合理设置 maxDays

```typescript
// 开发环境：较短
maxDays: 3

// 生产环境：根据合规要求
maxDays: 30
```

### 3. 监控磁盘空间

定期检查日志目录大小：
```bash
du -sh ~/.claude-chat/logs/
```
