✅ 设计决策

纯展示方案：移除所有需要后端支持的交互按钮，保留结构化卡片展示、状态徽章、语法高亮和原始 JSON 查看功能。此方案无需后端改动即可实施。

📊 方案对比
当前实现 vs 美化方案对比
特性 当前实现 美化方案
显示形式 简单文本行 结构化卡片
视觉层级 单层，无重点 标题 → 字段 → 描述 → 操作
状态指示 无 彩色徽章（待办/进行中/已完成）
类型识别 Emoji 前缀 左侧彩色条纹 + Emoji + 类型标签
交互操作 无 仅展示（无按钮交互）
原始 JSON 折叠/不显示 可选展开，带语法高亮
元信息 无 时间、分配人、标签等

✅ 纯展示方案总结

采用卡片化结构化展示，保留左侧类型色条作为视觉锚点。移除所有需后端支持的交互按钮，仅保留"查看原始 JSON"的展开/折叠功能（纯前端实现）。此方案无需后端改动，立即可实施。
