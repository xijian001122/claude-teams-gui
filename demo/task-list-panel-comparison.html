方案 1

可折叠面板

Sidebar 底部放置可折叠的任务面板，点击面板头部可展开/收起。适合需要常驻显示但不想占用太多空间的场景。

💡 点击任务面板头部可展开/收起，默认展开显示最多 5 个任务

方案 2

Tab 页切换

Sidebar 内容区域使用 Tab 切换团队列表和任务列表。充分利用现有空间，界面简洁清晰。

💡 点击 Tab 切换「团队」和「任务」两个视图，任务 Tab 显示完整任务信息

方案 3

悬浮抽屉

右下角悬浮按钮触发任务列表抽屉。最小化干扰，适合偶尔查看任务的场景。

💡 点击右下角悬浮按钮打开任务抽屉，再次点击或点击外部关闭

方案对比分析

方案 优点 缺点 适用场景
方案1: 可折叠面板
✓ 常驻显示任务数量
✓ 可快速展开查看
✓ 不离开当前团队视图
✗ 占用 Sidebar 底部空间
✗ 团队列表可能被挤压
任务较多且需要频繁查看的团队
方案2: Tab 页切换
✓ 界面简洁清晰
✓ 充分利用 Sidebar 空间
✓ 任务列表展示面积大
✓ 符合用户习惯
✗ 需要切换查看团队
任务与团队同等重要的场景
方案3: 悬浮抽屉
✓ 最小化界面干扰
✓ 随时可以打开
✓ 不占用 Sidebar 空间
✗ 需要额外点击打开
✗ 可能遮挡聊天内容
✗ 新任务通知不明显
偶尔查看任务的场景
⭐ 推荐方案：Tab 页切换（方案2）

推荐理由：

  • 界面最简洁，符合主流 IM 应用（如 Slack、Discord）的设计模式
  • 任务列表展示区域充足，可显示完整的任务信息（ID、名称、负责人、状态、依赖）
  • 用户学习成本低，Tab 切换是普遍认知的交互模式
  • 不占用额外的屏幕空间，Sidebar 区域被充分利用

实施建议：

  • 默认显示「团队」Tab，保持现有用户体验
  • 任务 Tab 添加未读/进行中的任务数量徽章
  • 点击任务可跳转或显示详情
  • 支持键盘快捷键切换 Tab（如 Ctrl+1/2）
