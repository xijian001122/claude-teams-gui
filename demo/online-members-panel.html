👥 在线成员列表面板 UI 设计方案

方案：Header 下拉面板
在 ChatArea Header 显示在线人数，点击展开成员列表下拉面板，按状态分组显示
team-lead 的团队
聊天消息区域 点击上方"2 执行中 · 1 空闲"可展开/收起面板
状态定义
三种成员状态及其视觉表现
状态 图标 颜色 定义 显示内容
执行中 红色 #ef4444 Agent 正在处理任务 状态徽章 + 任务描述
空闲 绿色 #22c55e Agent 没有在执行任务 最后活动时间
离线 浅灰 #d1d5db Agent 进程未运行 不显示在下拉面板中
状态转换逻辑
基于文件系统活动和消息流的自动状态检测
检测逻辑：
1. FileWatcher 检测到 inbox 有新消息
   → 该 Agent 标记为"执行中"
   → 广播 member_status 事件

2. 30秒内无新消息/活动
   → 自动转为"空闲"

3. Agent 进程停止
   → 标记为"离线"（不显示）
交互细节
下拉面板的交互行为和视觉反馈
交互 行为 视觉反馈
点击触发器 展开/收起面板 背景变蓝，箭头旋转
点击外部 自动关闭面板 面板淡出
悬停成员项 高亮显示 背景变为浅灰
状态更新 实时同步 数字动画更新

// Header 组件结构
<div class="chat-header">
  <div class="header-left">
    <div class="team-name">{team.displayName}</div>

    <!-- 在线人数触发器 -->
    <OnlineMembersTrigger
      busyCount={2}
      idleCount={1}
      onClick={togglePanel}
    />

    <!-- 下拉面板 -->
    {isOpen && (
      <OnlineMembersPanel
        members={members}
        onClose={closePanel}
      />
    )}
  </div>
</div>

⭐ 推荐方案
基于当前 Claude Chat 的设计，推荐 Header 下拉面板方案
推荐理由：
  • 与现有 ChatArea Header 设计兼容性好
  • 不占用聊天区域空间
  • 状态信息一目了然
  • 支持多标签页自动同步（通过 WebSocket）
  • 移动端和桌面端都能很好显示
