配置热加载 UI 设计方案

方案 1: 侧边栏指示器
当配置发生变更且需要重启时，侧边栏的"设置"标签显示红色徽章指示器。 用户可以点击设置进入查看详情，或悬停时显示快捷重启按钮。

变体 A: 红点徽章

💬 消息
👥 团队
⚙️ 设置 需重启
💡 点击"设置"标签查看效果

变体 B: 悬停显示重启按钮

💬 消息
👥 团队
⚙️ 设置
💡 悬停"设置"标签查看重启按钮
方案 2: 设置页面提示条
进入设置页面后，根据不同的配置状态显示相应的提示条。 包括：需要重启的警告条、已生效的成功提示、以及手动编辑检测的通知。
⚠️ 检测到配置变更，需要重启服务才能生效
常规
通知
高级
HTTP 服务监听端口
桌面通知 收到新消息时显示系统通知
消息提示音 收到消息时播放声音
💡 点击提示条上的按钮或开关组件进行交互
方案 3: 变更确认弹窗
当用户点击"重启"或"查看变更"时，显示详细的配置变更对比弹窗， 让用户清楚知道哪些配置被修改了，从什么值改为什么值。
⚠️ 配置有变更，需要重启

方案 优点 缺点
侧边栏指示器 • 用户一眼可见
• 不打扰正常操作
• 变体B可快捷重启 		• 需要进入侧边栏才看到
• 可能与其他通知混淆
设置页面提示 • 信息完整明确
• 操作按钮清晰可见
• 支持"稍后"选项 		• 需要主动进入设置页

变更详情弹窗 • 变更对比清晰
• 避免误操作
• 专业感强 		• 交互层级较深
• 变更多时阅读负担
推荐方案
组合方案：侧边栏指示器 + 设置页面提示 + 变更确认弹窗
  • 侧边栏：显示红点徽章"需重启"，让用户随时知道状态
  • 设置页：进入后顶部显示警告条，明确列出哪些配置需重启
  • 弹窗：点击重启按钮后显示变更确认，避免误操作
  • 需重启标记：表单字段旁显示"(需重启)"标签，提前告知用户
