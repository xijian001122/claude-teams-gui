# Claude Chat 需求文档

## 项目概述

Claude Chat 是一个为 Claude Code Teams 设计的可视化群聊界面，提供类似微信的聊天体验，支持实时消息同步、双向通信、数据持久化和主题切换。

---

## 功能需求

### 1. 核心功能

#### 1.1 团队管理
- **团队列表展示**: 左侧边栏显示所有团队，按最后活动时间排序
- **团队切换**: 点击团队名称切换当前聊天窗口
- **新建团队**: 支持创建新团队（快捷入口）
- **在线状态**: 显示团队成员在线/离线状态

#### 1.2 消息功能
- **群聊模式**: 团队内所有成员可见的群聊
- **私聊模式**: 支持 @成员 进行定向消息
- **消息发送**: 用户以 "user" 身份发送消息
- **消息接收**: 实时接收 Claude Teams 成员消息
- **历史消息**: 加载并显示历史聊天记录

#### 1.3 头像与身份
- **圆形头像**: 使用成员首字母作为头像（D/T/L/U）
- **头像颜色**: 不同角色固定配色
- **头像交互**: 点击头像自动 @该成员
- **身份标识**: 清晰区分 agent/user/system 消息

#### 1.4 @提及功能
- **输入框 @**: 输入 @ 弹出成员列表，支持键盘选择
- **头像 @**: 点击头像自动添加 @member 到输入框
- **高亮显示**: 被 @ 的消息特殊标记

### 2. 界面设计

#### 2.1 布局结构
```

│  Claude Teams                                    [◐] [设置] │
├────────────────┬────────────────────────────────────────────┤
│                │                                            │

│  ● dev-test    │                                            │
│    team        │  ┌───┐                                     │
│    3           │  │ D │ developer                           │
│                │  └───┘                                     │
│  ○ default     │  ┌─────────────────────────┐               │
│                │  │ 消息内容                 │               │
│  ───────────── │  └─────────────────────────┘               │
│                │                                            │
```

#### 2.2 主题系统
- **白天模式**: 浅色背景 (#FFFFFF)、深色文字 (#1F2937)
- **黑夜模式**: 深色背景 (#111827)、浅色文字 (#F9FAFB)
- **切换方式**: 点击顶部月亮/太阳图标切换
- **持久化**: 保存用户主题偏好

#### 2.3 头像配色

| 角色 | 首字母 | 背景色 |
|------|--------|--------|
| developer | D | #3B82F6 (蓝色) |
| tester | T | #10B981 (绿色) |
| team-lead | L | #8B5CF6 (紫色) |
| user | U | #F59E0B (橙色) |
| system | S | #6B7280 (灰色) |

#### 2.4 气泡样式
- **左侧气泡**: 其他成员消息，白色/深色背景
- **右侧气泡**: 用户自己消息，主题色背景
- **圆角**: 8px 圆角，指向发送者头像
- **间距**: 消息之间 8px，不同发送者之间 16px

### 3. 通知系统

#### 3.1 未读标记
- **团队列表**: 显示未读消息数量红点
- **消息预览**: 最新一条消息摘要
- **清除已读**: 进入团队自动标记已读

#### 3.2 桌面通知
- **新消息提醒**: 收到新消息时桌面通知
- **通知内容**: 显示发送者和消息摘要
- **点击行为**: 点击通知打开对应团队窗口
- **免打扰**: 可配置关闭桌面通知

### 4. 数据持久化

#### 4.1 存储策略
- **独立存储**: 与 Claude Teams 分离的持久化存储
- **双向同步**: 监听 Claude Teams 变化并同步
- **数据保留**: 默认保留 90 天，可配置

#### 4.2 归档功能
- **自动归档**: 团队删除时自动归档数据
- **归档查看**: 专门的归档入口查看历史团队
- **导出功能**: 支持导出聊天记录为 JSON/Markdown
- **手动删除**: 归档团队可手动永久删除

#### 4.3 数据清理
- **定时清理**: 每天凌晨 2 点自动清理过期数据
- **清理范围**: 超期消息、孤立附件、已归档团队
- **配置项**: 保留天数、清理时间、自动删除开关

### 5. 图片与附件

#### 5.1 图片处理
- **小图片** (< 500KB): Base64 存储在数据库
- **大图片** (> 500KB): 文件系统存储，数据库存路径
- **缩略图**: 自动生成 200x200 缩略图
- **点击查看**: 点击缩略图查看原图

#### 5.2 文件处理
- **文件类型**: 支持 PDF、ZIP、代码文件等
- **大小限制**: 单文件最大 10MB
- **存储方式**: 文件系统存储
- **下载功能**: 提供下载链接

### 6. 集成方式

#### 6.1 Claude Code 内置
```
/claude-teams-gui              # 启动（默认端口 3456）
/claude-teams-gui --port 8080  # 指定端口
```

#### 6.2 独立 CLI
```bash
npm install -g claude-teams-gui
claude-teams-gui               # 启动
claude-teams-gui --headless    # 仅启动服务器
```

## 非功能需求

### 性能要求
- **启动时间**: < 3 秒
- **消息延迟**: < 100ms（本地）
- **历史加载**: 支持懒加载，首次加载最近 50 条
- **内存占用**: < 200MB（空闲状态）

### 兼容性
- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **Node.js**: v18+
- **操作系统**: macOS、Linux、Windows

### 安全性
- **本地运行**: 仅监听 localhost，不暴露公网
- **数据隔离**: 每个用户独立数据目录
- **无远程传输**: 不发送数据到外部服务器

## 用户故事

### 故事 1: 查看团队对话
作为用户，我希望在浏览器中查看 Claude Teams 的实时对话，以便更直观地了解 agents 之间的协作过程。

**验收标准:**
- 打开界面自动加载所有团队
- 团队按最后活动时间排序
- 点击团队显示聊天记录
- 新消息实时出现在界面中

### 故事 2: 发送消息给 Agents
作为用户，我希望在聊天界面中直接发送消息给特定 agent 或整个团队，以便快速介入协作。

**验收标准:**
- 在输入框输入消息并发送
- 支持 @member 语法定向发送
- 点击头像自动 @该成员
- 消息显示在右侧气泡中

### 故事 3: 保留历史记录
作为用户，我希望即使 Claude Team 被删除，也能保留聊天记录，以便后续查阅。

**验收标准:**
- 数据存储在独立于 Claude 的目录
- 团队删除时自动归档
- 可在归档列表中查看历史团队
- 支持导出聊天记录

### 故事 4: 切换主题
作为用户，我希望在白天和黑夜主题间切换，以便在不同光线环境下舒适使用。

**验收标准:**
- 点击主题图标切换
- 切换即时生效
- 记住用户偏好
- 刷新页面保持主题
