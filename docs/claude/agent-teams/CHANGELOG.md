# Changelog

## [1.4.0] - 2026-03-14
### Added
- 新增"代码逻辑变更通知规范"（best-practices.md）
  - 当开发者修改核心业务逻辑时，必须主动通知测试者更新测试
  - 提供触发条件清单（5种场景）
  - 提供通知消息模板（包含5项必需内容）
  - 提供完整工作流程（从变更到测试更新）
  - 记录历史教训：2026-03-14 多位多车逻辑重构后测试失效的经验

### 经验教训
- **逻辑变更必须通知测试者**：开发者修改核心逻辑后，原有测试可能完全失效
- **通知内容要具体**：必须包含变更模块、原逻辑 vs 新逻辑、影响范围、测试文件、更新要点
- **测试者应更新而非删除**：收到通知后应更新测试用例，而不是直接删除
- **历史案例**：多位多车入场逻辑从多参数改为 Bo 对象后，所有相关测试都需要重写
